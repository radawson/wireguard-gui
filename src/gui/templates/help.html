Help: WireGuard concepts and terms

Peer

A peer is one participant in a WireGuard VPN: a device or node with its own keypair and an address on the tunnel. In this app, the Peers section lists the devices you add to your networks. Each peer has a private key (and derived public key), a VPN IP, optional endpoint and listen port, and belongs to one network. You can mark one peer per network as the lighthouse (the reachable server others connect to).

Network

A network in this app is one WireGuard VPN (one tunnel). It is defined by a name, a base IP and subnet (e.g. 10.10.11.0/24), a host adapter name, DNS, and options like allowed IPs and persistent keep-alive. Peers belong to a network; the network is tied to a single WireGuard interface (adapter) on the server.

Adapter (Host Adapter Name)

The adapter (or host adapter name) is the kernel interface name for the WireGuard tunnel on this server, e.g. wg0 or wg1. One network maps to one adapter. Commands like wg-quick up wg0 bring that tunnel up. When you choose "Host Adapter Name" in the network form, you are selecting which interface this network uses.

Lighthouse

In this app, a lighthouse is the peer that acts as the reachable server for a network: it has a public endpoint (IP and listen port) and stays online so other peers can connect to it. You designate one peer per network as the lighthouse; other peers use it as their single [Peer] endpoint in their generated config.

Private key & Public key

WireGuard uses a private key (secret, never shared) and a public key (derived from it, shared with other peers). Each peer and each network interface has its own keypair. The app can generate a private key for you; the public key is shown as "derived" and is what you put in other peers' [Peer] stanzas.

Allowed IPs

Allowed IPs is a comma-separated list of CIDR ranges (e.g. 10.10.11.0/24) that tell WireGuard which traffic to send over the tunnel for that peer. It acts as a routing rule: only traffic destined for those ranges goes through the VPN. For a full tunnel, you typically use the network's subnet.

Endpoint

The endpoint is the reachable address of the other side of the tunnel: host (or IP) plus port. In this app it appears as "Endpoint IP" and "Listen Port" on a peer. For client peers, the endpoint is usually the lighthouse's public IP and listen port so they know where to connect.

Preshared key

A preshared key (PSK) is an optional extra symmetric key used between one pair of peers. WireGuard supports it for additional security (e.g. post-quantum). In the peer form you can leave it blank or generate one; if set, it is included in the generated config.

Persistent keep-alive

Persistent keep-alive is an interval in seconds. When set, the peer sends a small packet at that interval so that NATs and firewalls keep the path open. Use it for peers behind NAT (e.g. laptops or home clients) so the server can reach them. A common value is 25.

VPN IP & Base IP

VPN IP (or "Address" in config) is the IP address assigned to this peer on the tunnel (e.g. 10.10.11.5). Base IP is the network's base range (e.g. 10.10.11.0) and the subnet (e.g. /24) defines how many addresses the network has. The app uses the base IP and subnet to offer you valid VPN IPs when adding a peer.
